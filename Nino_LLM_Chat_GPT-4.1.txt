1. Predictive Modeling  
Description: Build a model to predict future water flow rates or pressures at specific borehole locations using historical sensor data.  
Evaluation:  
• Root Mean Squared Error (RMSE) or Mean Absolute Error (MAE) on a held-out test set  
• Ability to generalize to unseen time periods  
• Robustness to missing or noisy data
 
---
 
2. Anomaly Detection  
Description: Develop an algorithm to detect anomalies or outliers in flow, pressure, or temperature data that may indicate sensor faults or unexpected subsurface events.  
Evaluation:  
• Precision, recall, and F1-score for labeled anomalies (if available)  
• Clarity and interpretability of detected anomalies  
• False positive/negative rates
 
---
 
3. Phase Classification  
Description: Classify each hourly record as belonging to the “hot water injection” or “ambient temperature circulation” phase using only the sensor data (not the timestamp).  
Evaluation:  
• Classification accuracy, precision, recall, and F1-score  
• Confusion matrix analysis  
• Model interpretability (feature importance)
 
---
 
4. Feature Importance & Interpretation  
Description: Identify which sensor measurements are most important for predicting system performance or phase transitions.  
Evaluation:  
• Clarity and justification of feature ranking  
• Use of appropriate techniques (e.g., SHAP, permutation importance)  
• Insights provided for system operation
 
---
 
5. Event Impact Analysis  
Description: Quantify the impact of switching from hot water injection to ambient circulation on key system variables (e.g., pressure, flow, EC).  
Evaluation:  
• Statistical significance of detected changes  
• Visualization of before/after effects  
• Quality of causal inference or change-point detection
 
---
 
6. Data Imputation  
Description: Develop algorithms to fill in missing or corrupted sensor readings using the available multivariate time series.  
Evaluation:  
• Imputation accuracy (e.g., RMSE/MAE on artificially masked data)  
• Preservation of temporal and cross-sensor correlations  
• Robustness to different missing data patterns
 
---
 
7. Multivariate Time Series Forecasting  
Description: Build a model to jointly forecast multiple variables (flow, pressure, EC) at several depths for the next 24 hours.  
Evaluation:  
• Multivariate RMSE/MAE  
• Ability to capture dependencies between variables  
• Performance on both short-term and long-term horizons
 
---
 
8. Root Cause Analysis  
Description: Given a detected anomaly, infer the most likely cause (e.g., equipment malfunction, phase change, external disturbance).  
Evaluation:  
• Correctness of root cause identification (if ground truth is available)  
• Clarity and interpretability of reasoning  
• Use of supporting evidence from the data
