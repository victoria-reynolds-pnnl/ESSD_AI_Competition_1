**Week 2: Data Preparation Approach**

**Data Review Approach:**

The raw data set consists of 72 CSV files that list cold snap and heat
wave events in the continental US from 1980-2004. To simplify our
analysis, we opted to use only a subset of the data. In the original
data set, three averaging methods were used to compile temperature
events (simple mean, area-weighted mean, and population-weighted mean).
We chose to use only population-weighted averages in our analysis
because we are focused on potential power-grid impacts, and population
is a key driver for power demands. Additionally, there are 12
definitions for temperature events, and we chose to limit our analysis
to definition #9 (daily minimum temperature \>95^th^ percentile for heat
waves or \<5^th^ percentile for cold snaps with minimum duration of 2
days). This allowed us to focus on temperature extremes (highs and lows)
while also working with a larger number of rows. Our data filtering
reduced the number of CSV files in the original data set from 72 to 2
(one for cold snaps with population-weighted average and an analogous
file for heat wave events).

We have uploaded the relevant files for our ML approach; all datasets
are available [here.](https://data.pnnl.gov/group/nodes/dataset/34393)

**Data cleaning** involved standardizing variable names across event
types (renaming highest_temperature and lowest_temperature to a common
extr_temp field), tagging each record with its event type (heat_wave or
cold_snap), and assigning unique row identifiers (perm_num).

**ML algorithm approaches:**

Our ML approach uses a two-step framework: (1) cluster analysis to
capture geographic structure and compare patterns across regions, and
(2) a regression model that includes region as a fixed effect (with
potential random effects to be determined). Depending on the outcome and
research question, the regression may be specified as a count model
(Poisson or negative binomial), beta regression, or another appropriate
model type.

**Feature engineering** included extracting the event start year
(start_yr) from the date field, computing an impact metric as the
product of spatial_coverage, duration, and absolute departure of
extr_temp from 290K (representing intensity for heat wave and cold snap
events), and creating region-specific quartiles for both duration and
extreme temperature within each NERC subregion and event type. These
quartiles were then combined into a composite categorical variable
(extreme_type) encoding relative severity (e.g., heat_dur_temp34 for a
heat wave in the 3rd duration quartile and 4th temperature quartile). A
summary table aggregating event counts by region, year, and extreme type
was also generated. For the impact feature, timeseries tables were
generated by region to enable detecting event co-occurrence.

**Normalization:**

We have normalized the extreme quartiles for duration and temperature by
region and type. We will normalize impact variable during data analysis
depending on approach decided.

**AI\'s role** was central to the data wrangling and feature engineering
steps: the code for reading, combining, cleaning, renaming, and creating
derived features (lines 17--101 in data_clean.R and lines 21--190 in
extreme_DD.R) was generated using GPT-5.2 via Model Garden, with
human-written comments added afterward for documentation. Also,
Microsoft Copilot assists in code writing by providing potential
suggestions. The setup/configuration code and final file outputs were
written manually.
