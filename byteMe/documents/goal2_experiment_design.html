Experiment overview
Predict temperature at production wells TL and TN up to 24 hours ahead, given the history of hot-water injection at TC. Four prediction horizons are treated as separate regression tasks. Success requires beating a strong persistence baseline on all horizons and passing six physical-validity checks.
Scientific question
Can a data-driven model learn the thermal transport relationship between the injection well (TC) and the production wells (TL, TN) well enough to predict breakthrough timing, peak temperature attenuation, and the shape of the temperature rise curve — without access to rock thermal conductivity or fracture geometry?
Null hypothesis to falsify
A model that knows only the current temperature at each production well (persistence) produces forecasts that are as accurate as any model using injection history and flow rate features. Failing to reject this null at h ≥ 60 min means the model has learned nothing about transport physics.

Key constraints
Temporal ordering
No shuffling of any kind. All train/test splits are strictly chronological. Expanding-window CV only.
No future leakage
Features at time t may only use data from t and earlier. Lag-window rows that cross a phase boundary must be dropped.
Physical bounds
All predictions must lie within [T₀ − 0.5°C, T_injection_max + 0.5°C]. Apply post-hoc clipping and report the clip rate.

Total feature count summary
Family A: 24 features · Family B: 19 features (per well) · Family C: 15 features · Family D: 8 features · Family E: 4 features = ~70 features per target well. This is a manageable size for Random Forest and Lasso. For LSTM, reduce to ~30 by using only the most physically motivated lags (marked Critical/High above).
Cross-well contamination rule: When predicting TL temperature, do not include TN temperature lags from the same time window — and vice versa. TL and TN respond to the same injection and are correlated; including contemporaneous TN lags when predicting TL allows the model to cheat via a proxy that contains future information about the shared thermal signal. Only include the other production well's temperature if it is lagged by at least h minutes (the prediction horizon).

Five models are fitted per (well, horizon) pair. All share the same feature store and train/test boundaries. Models are fitted independently — no ensembling until the ablation study confirms it adds value.

Physics-informedExponential approach model (EAM)
Fits a parametric thermal response curve to the training set: ΔT(t) = A × (1 − exp(−t / τ)) × f(Flow) where A = asymptotic temperature rise, τ = thermal time constant, and f(Flow) is a flow-rate scaling factor. Parameters A, τ, and the flow exponent are estimated by nonlinear least squares (scipy.optimize.curve_fit). The EAM residuals are then used as features in the RF model (hybrid variant). Also serves as a standalone baseline that embeds physically correct functional form.

TL and TN; report A and τ per well — these are geological outputs

Evaluation protocol
All metrics computed on the held-out test set (last 22 days of hot injection phase). The test set is touched exactly once per model, after all hyperparameter decisions are frozen. Results are reported across three stratifications: full test period, breakthrough window, and steady-state window.
Layer 1 — Predictive accuracy metrics (all models, all horizons)
RMSE per (well, horizon)
√[ mean( (ΔT_pred − ΔT_true)² ) ] on full test set
Report as 2×4 matrix (wells × horizons). Primary numerical output.
Report absolute value in °C. Compare across models.
Skill score SS(h)
1 − RMSE_model / RMSE_persistence, per horizon
Positive = beats persistence. Negative = worse than doing nothing.
Target: SS > 0.10 at h=60 min; > 0.20 at h=240 min
Stratified MAE
MAE computed separately on breakthrough window (dΔT/dt > 0.01°C/min sustained > 30 min) and steady-state window
Detects models that perform well overall by excelling on easy steady-state while failing during the critical breakthrough event.
MAE_breakthrough must be ≤ 2× MAE_steady_state. Larger ratio = model fails when it matters.
Physical bounds violation rate
Fraction of predictions outside [T₀ − 0.5, T_inject_max + 0.5]
Count rows violating bounds; report as % of test set per model.
Must be 0%. Apply post-hoc clipping; report clip rate separately.
MAE vs climatological baseline
Compute separately: does the model beat the mean-curve baseline?
A model that doesn't beat the climatological curve has learned only the average trend, not current-state-sensitive transport.
Must beat climatological baseline at h=240 min by > 10%.
Layer 2 — Physical validity checks (primary model RF-Lag + LSTM)
Breakthrough onset timing error
Define onset: first ΔT > 2°C sustained > 30 min. Compare true onset time to earliest time the h=240 min model first predicts ΔT > 2°C.
Measures whether the model can anticipate the thermal front arrival, not just react to it after the fact.
Target: |error| < 3 hrs at h=240 min horizon.
Peak attenuation factor error
α_true = ΔT_prod_peak / ΔT_inject_peak. α_pred = predicted_peak / ΔT_inject_peak. Report (α_pred − α_true) / α_true.
Tests whether the model has learned correct heat loss to the rock matrix en route.
Target: fractional error < 15%.
SHAP lag vs cross-corr lag
Extract lag of highest mean |SHAP| for TC_INT_mean_lag_k features. Compute cross-correlation peak lag τ* between TC injection temp and TL/TN response. Compare.
If the model relies on a 5-min lag but τ* = 4 hours, it has found spurious correlation, not the thermal signal.
|SHAP-dominant lag − τ*| < 20% of τ*.
Flow-rate sensitivity ratio
Segment test period: top 25% Net Flow rows vs bottom 25%. Compute mean predicted dΔT/dt (°C/min) in each segment.
Physical expectation: higher flow → faster thermal transport → steeper predicted temperature rise.
Ratio (high-flow dΔT/dt) / (low-flow dΔT/dt) > 1.3.
TL vs TN forecast differentiation
Pearson correlation between predicted TL series and predicted TN series on test set at h=240 min.
Spatially distinct wells should show distinct predictions. Near-identical forecasts indicate the model learned a shared signal, not well-specific transport.
Correlation < 0.90 at h=240 min.
Cumulative heat monotonicity
Plot cumulative_heat_input (x) vs predicted ΔT at h=240 min (y) for test set. Fit isotonic regression. Count monotonicity violations.
More injected heat should not produce lower predicted temperatures under steady injection.
< 5% of test-set rows violate monotonicity during non-shutdown periods.

Controlled experiments that isolate the contribution of individual design decisions. Each ablation changes exactly one thing relative to the primary RF-Lag model. Run on the full train/test split — not just CV — and report skill score delta vs the full model.

Adds EAM-predicted ΔT and residual as extra features. Measures whether the physics prior improves accuracy, especially on the physical validity checks.

Primary model

Interpretation key: If A2 (removing flow) causes a large skill score drop at h=240 min, the model is correctly using advective transport speed. If A4 (removing elapsed time) causes a larger drop than A2, the model is primarily a trend learner. If A5 shows that short lookback performs nearly as well as long lookback, the thermal memory in the extended lags is not adding signal — this is a data limitation finding worth reporting.
Step-by-step run order
All steps in sequence. Steps 1–6 are data and feature preparation. Steps 7–11 are model fitting and evaluation. Steps 12–15 are analysis and reporting. Only Step 13 (test set evaluation) is irreversible — everything before it can be iterated.
1
Compute T₀ per well
Load the pre-injection segment (Dec 10–13). For each well, compute T₀ = median of (TEC-INT-U + TEC-INT-L)/2 over this window. Store as a constant per well. This is never recomputed — it anchors all ΔT calculations.
2
Isolate hot injection segment and identify ramp-up
Slice from injection start timestamp to phase 1 end. Identify the ramp-up window: first row where Net Flow > 0.5 L/min to first row where Net Flow has been > 80% of its 6-hr rolling max for 60 consecutive minutes. Exclude the ramp-up rows, log their count.
3
Check for sensor drift in monitor wells
Plot TU-TEC-INT-U and TS-TEC-INT-U over the full hot phase. Fit a linear trend. If slope > 0.002°C/min (0.12°C/hr) in a monitor well far from TC, apply linear detrend to that well's temperature columns before feature construction.
4
Construct all features and targets
Apply lag construction, rolling windows, cumulative heat integral, elapsed time, and ΔT gradient. Compute delta_T_above_T0 for all wells. Apply the lag-boundary exclusion rule (drop first W rows post-ramp-up). Produce a single flat feature matrix — one row per minute — with 70 features and 8 target columns (2 wells × 4 horizons). Save this as the canonical feature store.
5
Apply train/test split and compute cross-corr lag τ*
Split at day-50 boundary. Apply lag-boundary exclusion at the split point. Compute the cross-correlation between TC_INT_mean and each production well's delta_T on the training set only. Record the peak lag τ* for TL and TN — this is used later as ground truth for the SHAP lag validity check.
6
Fit persistence and climatological baselines
Compute persistence RMSE for all 8 targets on the test set. Fit the climatological mean curve (60-min bins, 5-bin smoothed) from the training set. Evaluate on test set. Record baseline RMSE values — these are the denominators for all skill scores computed later.
7
Fit EAM (exponential approach model)
Run nonlinear least squares on training set for TL and TN. Record A, τ, and flow exponent β. These are scientific outputs independent of the ML performance. Add EAM residuals as features to the feature store (for the hybrid variant). Plot the EAM fit overlaid on the actual training temperature curve to visually confirm fit quality.
8
Hyperparameter search for RF-Lag via expanding-window CV
Grid search over {max_features, min_samples_leaf} using 5 expanding-window folds. Optimise for mean RMSE at h=240 min (primary horizon). Apply the same hyperparameters for all horizons and both wells to prevent per-target overfitting. Log best parameters and CV score.
9
Fit all ablation variants (A1–A7)
Using the CV-selected hyperparameters, retrain the RF with each ablated feature set on the full training set. Record training-set RMSE for each variant (sanity check: should match CV results approximately). Do not evaluate on the test set yet.
10
Train LSTM
Train with early stopping using fold-5 as the validation window. Log training curves. If validation loss diverges after epoch 20 without improvement, reduce hidden units or increase dropout. Save the best checkpoint (lowest validation RMSE).
11
Freeze all models — compute SHAP on training set
Run SHAP TreeExplainer on the RF-Lag model using the training set. Record the mean |SHAP| per feature. Identify the lag value k of the TC_INT_mean_lag_k feature with highest mean |SHAP|. Compare to τ* from Step 5. Log the discrepancy. This is done before test-set evaluation so the physical validity check is not influenced by test outcomes.
12
Test set evaluation — run once, record everything
Apply all frozen models to the test set. Compute all Layer 1 and Layer 2 metrics. Check physical bounds violations and apply clipping. Record stratified MAE for breakthrough vs steady-state windows. Do not re-train any model based on test results — this step is final.
13
Ablation delta analysis
Compute skill score delta = SS(full model) − SS(ablated model) for each of A1–A7 at h=240 min. Rank ablations by impact. The ranking answers: which feature family carries the most predictive value? Which design choice (ΔT vs raw T, long vs short lookback) had the largest effect?
14
Sensitivity analysis on T₀ and breakthrough onset criterion
Recompute breakthrough onset timing error using three alternative T₀ definitions (mean, 10th percentile, first-hour median) and two onset thresholds (ΔT > 1°C, ΔT > 3°C). Report the range of timing error values. This quantifies how much the physical validity conclusions depend on definitional choices.
15
Compile final outputs
Produce the 8-cell RMSE matrix, skill score matrix, physical validity scorecard, ablation ranking, EAM-fitted parameters (A, τ per well), SHAP lag plot vs τ*, and flow-rate sensitivity ratio. These together constitute the full experiment result.

Recommended reporting standard: Report every check in the table, whether passed or failed. A paper or report that shows skill score without the flow-rate sensitivity check, or RMSE without the breakthrough-window stratification, is presenting an incomplete and potentially misleading picture of model quality. The physical validity checks are not optional extras — they are the primary scientific contribution of this experiment.
